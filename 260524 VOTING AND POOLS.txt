Short answer: no, they don't always match directly. They live at different stages and have different semantics.
pk_state is per-pool per-bar:

±1 = divergence (line slope opposite to price slope)
±2 = PM sentinel (line and price agree, large slope — "trend continuation")
0 = neutral

pk_raw is the vote machine's output after combining the two pools:

±1 = vote machine cleared threshold in that direction
0 = vote machine didn't clear threshold

So pk_state is RAW, pk_raw is COOKED.
They diverge in important ways:

PM sentinels are routed to neutral in the vote. So if pool_close fires pk_state=+2 (trend long, PM) and pool_wide is neutral, pk_raw might be 0 (no directional vote) even though one pool's state has a long-leaning signal.
Vote machine subtracts opposing PM weights (pm_suppress_str). A pool firing pk_state=-2 (PM short) can reduce the long bucket score below threshold even though the OTHER pool fired pk_state=+1.
Two pools firing opposite directions cancel via vote ratios. pk_state=+1 in close + pk_state=-1 in wide → vote ratios both fall below threshold → pk_raw=0.

f_bb(src, len, mult) =>
    basis = ta.sma(src, len)
    dev   = mult * ta.stdev(src, len)
    upper = basis + dev
    lower = basis - dev
    pct   = (src - lower) / (upper - lower)
    (ob - os) * pct + os

    r   = ta.rsi(src, rsi_len)
    raw = ta.stoch(r, r, r, stc_len)

// Converts f_pk_state return value into [long, short, neutral] weight buckets.
// PM sentinels (PK_PM_LONG = 2.0, PK_PM_SHORT = -2.0) are weighted at t_pm_dilution.

// via the sentinel mechanism. The two systems are intentionally decoupled.

f_vote(state, weight) =>
    // PM sentinels (±2.0) produce no directional vote — treated as neutral
    // Suppression of opposing bucket handled in post-processing block
    if na(state)
        [0.0, 0.0, 0.0]
    else if state == PK_PM_LONG or state == PK_PM_SHORT
        [0.0, 0.0, float(weight)]
    else if state > 0
        [float(weight), 0.0, 0.0]
    else if state < 0
        [0.0, float(weight), 0.0]
    else
        [0.0, 0.0, float(weight)]
